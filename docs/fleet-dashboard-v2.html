Fleet Dashboard v2 tracker #1

A cross-product view — product → repo → worktree — with two-level Kanban, worktree/PR health, and plan↔work links. Shipped as four leaves (#2–#5) in fleet-dashboard/.

One morning-triage page showing what's in flight, what's merged-but-not-cleaned, what's stale, and which plan cards have a worktree. Forge-agnostic and config-driven so it runs on a laptop against local checkouts or as a scheduled cloud job with no checkouts at all.

The pipeline

config (fleet.config.json) ─┐
  collect_kanban() ──── plan cards (product coordinator + per-repo plans)
  worktree walk ─────── git/PR/flag health (squash-merge-aware)
  build_product_tree() ─ group repos→products, worktrees→repos (+ unaffiliated)
  link_worktrees_to_cards() ─ pair build-<slug> / <repo>-<slug> ↔ card by naming
              ↓
status.json ──▶ self-contained dashboard.html (data inlined, no CORS)
