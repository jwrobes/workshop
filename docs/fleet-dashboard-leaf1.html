How the seam works

The collector body never calls gh directly. It picks a forge from config (make_forge(cfg["forge"])) and asks the interface for data:

  • Forge.list_prs(slug, branch) → PR dicts; the collector ranks them with pick_pr() (prefer merged > open > closed), preserving the v1 selection.
  • Forge.list_repos(product) → member repos (used by the product spine in #4).
  • Swapping to GitLab later is one class, zero collector edits.

Reviewer notes

Offline unprotected is approximate. Under --no-gh the collector can't know PR state, so a dirty/ahead-unmerged worktree may be flagged unprotected even when an open PR would protect it. This matches the v1 engine; it's documented rather than changed so the port stays faithful. Say the word if you'd prefer a tri-state ("PR unknown") instead.
